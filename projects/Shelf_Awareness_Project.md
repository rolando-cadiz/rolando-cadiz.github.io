Shelf Awareness is a pantry inventory management system designed to help users keep track of ingredients, manage recipes, and make better use of the food they already have. The application allows users to maintain a digital pantry, view recipe options, and interact with recipes through features such as ratings and sorting. 

This project was created as part of my ICS 414 software engineering group project. Unlike smaller programming assignments, Shelf Awareness required us to work as a team on a larger full-stack application. Through this project, I gained experience with frontend development, backend API logic, database interaction, debugging, and collaborative software development.

[GitHub Repository](https://github.com/shelf-awareness)

[Vercel Deployment](https://shelf-awareness.vercel.app/)

Personal duties on this project included:

- Leading a UI overhaul and rebrand to improve the overall look, feel, and usability of the application.

- Designing and implementing the recipe rating feature, allowing users to rate recipes and see rating data reflected in the application.

- Creating the logic for recipe sorting, making it easier for users to organize and browse recipes.

- Working with frontend components, API routes, and database logic to connect user interactions with stored application data.

- Debugging and fixing issues as they came up, especially problems involving how the frontend, backend, and database worked together.

- Collaborating with teammates to troubleshoot application errors and keep the project moving forward.

## Reflection

One of the most valuable parts of this project was building the recipe rating system. This feature helped me better understand how a full-stack application works because it required multiple parts of the system to communicate correctly. A user action on the frontend had to be sent through an API, processed correctly, stored in the database, and then reflected back in the user interface.

The biggest challenge I faced was time management. I had a personal goal of closing two to four issues per week, but I also had to balance other classes and work. This taught me the importance of consistency, planning, and communication when working on a team-based software project.

Overall, Shelf Awareness helped me grow as both a developer and a teammate. It gave me practical experience with collaborative software engineering and helped me understand how real web applications are built, debugged, and improved over time.
